# Fix Registration Success/Failure Logic

The registration process incorrectly shows a "failed" alert even when the API returns a successful response (statusCode: 200, succeeded: true). This is caused by a crash in `AuthService.register` when attempting to access `response.data.refreshToken` if `data` is missing from the response. Additionally, the frontend does not check the `succeeded` flag of the `ApiResponse`, potentially leading to false success messages for 200 OK responses that indicate failure.

## Steps

### 1. Fix `AuthService.register` in `src/app/Services/auth.service.ts`
- Modify the `tap` operator to safely check for the existence of `response.data` before attempting to access `refreshToken` and set tokens.
- Ensure that if `response.data` is missing, the process doesn't throw an error if `response.succeeded` is true.

### 2. Fix `RegisterComponent.onSubmit` in `src/app/Core/register-component/register-component.ts`
- Update the `subscribe` block to handle the `ApiResponse` in the `next` callback.
- Check `response.succeeded` to determine whether to show the success alert or the failure alert.
- If `response.succeeded` is false, show an alert with `response.message`.

## Validation
- Verify that registration with a successful response (even without `data`) shows the "Registration successful!" alert.
- Verify that registration with a response where `succeeded: false` shows the appropriate error message.
- Verify that actual HTTP errors (e.g., 500 Internal Server Error) are still handled by the `error` block.
