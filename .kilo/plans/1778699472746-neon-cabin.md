# Fix 404 Not Found on Refresh in Production (MonsterASP / IIS)

The user is experiencing "Not Found" (404) errors when refreshing the page on any route other than the root in the production environment hosted on MonsterASP (which uses IIS). This is because the server is trying to find a physical file at the requested URL instead of delegating the routing to the Angular application.

## Analysis
Angular uses `PathLocationStrategy` by default. On a page refresh, the browser requests the current URL from the server. On IIS, if the requested URL doesn't correspond to a physical file or directory, it returns a 404. A `web.config` file is needed to configure the IIS Rewrite module to redirect all non-file requests to `index.html`.

## Implementation Plan
Since the project has a `public` folder configured as an asset source in `angular.json`, creating a `web.config` file in that directory will ensure it is copied to the root of the production build.

1. Create a `web.config` file in the `public` directory with the appropriate IIS rewrite rules for Single Page Applications (SPAs).
