# API Documentation Plan for Angular Integration

## Overview
Create comprehensive API documentation for the SchoolProject .NET Core Web API to enable seamless integration with an Angular frontend application.

## Current API State Analysis

### Implemented Controllers & Endpoints (4 controllers, 23 endpoints)

**1. AuthenticationController** (7 endpoints) - Auth & JWT
- POST `/api/Authentication/SignIn` - User sign-in
- POST `/api/Authentication/Refresh-Token` - Refresh JWT token
- GET `/api/Authentication/Validate-Token` - Validate token
- GET `/api/Authentication/ConfirmEmail` - Email confirmation
- POST `/api/Authentication/SendResetPasswordCode` - Request password reset
- GET `/api/Authentication/ConfirmResetPasswordCode` - Confirm reset code
- POST `/api/Authentication/ResetPassword` - Reset password

**2. AuthorizationController** (9 endpoints) - Role & Claim management (Admin only)
- POST `/api/AuthorizationRouting/Roles/Create` - Create role
- POST `/api/AuthorizationRouting/Roles/Edit` - Edit role
- DELETE `/api/AuthorizationRouting/Roles/Delete/{id}` - Delete role
- GET `/api/AuthorizationRouting/Roles/Role-List` - List all roles
- GET `/api/AuthorizationRouting/Roles/Role-By-Id/{id}` - Get role by ID
- GET `/api/AuthorizationRouting/Roles/Manage-User-Roles/{userId}` - Get user roles
- PUT `/api/AuthorizationRouting/Roles/Update-User-Roles` - Update user roles
- GET `/api/AuthorizationRouting/Claims/Manage-User-Claims/{userId}` - Get user claims
- PUT `/api/AuthorizationRouting/Claims/Update-User-Claims` - Update user claims

**3. ApplicationUserController** (6 endpoints) - User management (Admin/User)
- POST `/api/User/Create` - Create user
- GET `/api/User/Paginated` - Paginated user list
- GET `/api/User/{id}` - Get user by ID
- PUT `/api/User/Edit` - Edit user
- DELETE `/api/User/{id}` - Delete user
- PUT `/api/User/Change-Password` - Change user password

**4. EmailsController** (1 endpoint) - Send emails (Admin/User)
- POST `/api/EmailsRoute/SendEmail` - Send email

### Missing Controllers (Handlers exist but no API)
- **StudentController** - Would need 6 endpoints (CRUD + paginated list)
- **DepartmentController** - Would need 3 query endpoints
- **InstructorController** - Would need 2 endpoints

### API Response Format
All endpoints return `Response<T>` wrapper:
```json
{
  "succeeded": true,
  "message": "Success message",
  "data": { ... },
  "errors": null,
  "statusCode": 200
}
```

### Authentication
- JWT-based authentication with access + refresh tokens
- Role-based authorization (Admin, User roles)
- Use Bearer token in `Authorization` header
- Token refresh flow required

## Documentation Deliverables

### 1. OpenAPI/Swagger Specification
- Generate comprehensive OpenAPI 3.0 spec (YAML/JSON)
- Include all request/response schemas
- Document headers, query parameters, request bodies
- Add examples for each endpoint
- Include authentication/authorization details
- Export to `api-docs/swagger.yaml` or similar

### 2. Angular Integration Guide
- **Authentication Setup**
  - How to obtain JWT tokens (sign-in)
  - Token storage strategy (localStorage vs sessionStorage vs cookies)
  - Attaching Bearer token to requests
  - Token refresh implementation
  - Token expiration handling
  - Logout flow

- **HTTP Client Configuration**
  - Create Angular service for API calls
  - Interceptor for adding Authorization headers
  - Error handling interceptor
  - Base URL configuration

- **Environment Setup**
  - API URL configuration
  - Environment-specific endpoints

### 3. API Endpoint Reference
Organized by feature area:
- Authentication & Authorization (detailed flow diagrams)
- User Management
- Email Services
- (Optional) Student/Department/Instructor if controllers are to be implemented

Each endpoint documentation:
- HTTP method & full URL
- Required roles/permissions
- Request parameters (body, query, route)
- Request example (JSON)
- Response structure with examples
- Error responses with status codes
- Notes on special handling

### 4. Data Models Reference
TypeScript interfaces for all request/response DTOs:
- Authentication DTOs (SignInRequest, TokenResponse, etc.)
- User DTOs (UserCreate, UserResponse, etc.)
- Role/Claim DTOs
- Email DTOs
- Pagination types

### 5. Angular Services & Components Patterns
- Service structure examples (one service per feature)
- Observable patterns with RxJS
- Error handling with catchError
- Loading states
- Form validation examples (reactive forms)
- Guard implementations (auth guards, role guards)

### 6. Common Operations Guide
- **Login flow**: Sign in → store tokens → navigate
- **Protected routes**: Using AuthGuard, role-based routing
- **Refresh token flow**: Intercept 401 → refresh → retry
- **Form handling**: Validation patterns, submission
- **Pagination**: Query parameters, paginator component

### 7. CORS & Network Configuration
- Current CORS settings (allow any origin - may need production config)
- Allowed headers/methods
- HTTPS requirements
