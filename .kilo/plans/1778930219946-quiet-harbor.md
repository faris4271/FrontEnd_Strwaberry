# Fix Loading State in Forgot Password Component

The user reported that the "Forget Password" page continues to show a loading state after sending an email until the user interacts with the environment. This is a strong indicator of an Angular change detection issue where the UI is not automatically updating when the asynchronous request completes.

## Analysis
In `ForgotPasswordComponent`, the `isLoading` property is set to `true` before starting the request and set to `false` in both the `next` and `error` callbacks of the `authService` observables. However, the UI does not seem to reflect this change immediately.

Since the component uses the default change detection strategy and is a top-level route, this behavior is unexpected but can be resolved by explicitly triggering change detection using `ChangeDetectorRef`.

## Proposed Changes
- Inject `ChangeDetectorRef` into `ForgotPasswordComponent`.
- Call `this.cdr.detectChanges()` after updating `isLoading` and other state variables (`step`, `message`, `isError`) in the `sendCode` and `confirmCode` methods.

## Implementation Steps
1.  Modify `src/app/Core/forgot-password-component/forgot-password-component.ts`:
    - Import `ChangeDetectorRef` from `@angular/core`.
    - Inject `ChangeDetectorRef` using `inject(ChangeDetectorRef)`.
    - In `sendCode()`:
        - In `next` callback: call `this.cdr.detectChanges()` after setting `this.isLoading = false`.
        - In `error` callback: call `this.cdr.detectChanges()` after setting `this.isLoading = false`.
    - In `confirmCode()`:
        - In `next` callback: call `this.cdr.detectChanges()` after setting `this.isLoading = false`.
        - In `error` callback: call `this.cdr.detectChanges()` after setting `this.isLoading = false`.
