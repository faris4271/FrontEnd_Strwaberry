# Add Arabic Language Support

This plan outlines the steps to implement Arabic language support (and English) for the website using `ngx-translate` and RTL (Right-to-Left) layout support, without requiring a backend.

## Goals
- Implement multi-language support (English and Arabic).
- Enable runtime language switching.
- Ensure the layout mirrors correctly for Arabic (RTL).
- Maintain a frontend-only implementation.

## Technical Approach
- Use `@ngx-translate/core` and `@ngx-translate/http-loader` for translation management.
- Store translations in JSON files within `src/assets/i18n/`.
- Manipulate the `dir` attribute of the `<html>` element to handle RTL/LTR switching.
- Update Tailwind CSS classes to use logical properties (e.g., `start`, `end` instead of `left`, `right`) where necessary to ensure layout consistency across languages.

## Implementation Steps

### 1. Setup and Configuration
- [ ] Install `@ngx-translate/core` and `@ngx-translate/http-loader`.
- [ ] Configure `TranslateModule` in `src/app/app.config.ts` using `TranslateHttpLoader`.
- [ ] Create translation directories: `src/assets/i18n/`.
- [ ] Create `src/assets/i18n/en.json` with all extracted English strings.
- [ ] Create `src/assets/i18n/ar.json` with the corresponding Arabic translations.

### 2. Template and Component Translation
- [ ] Replace hardcoded text in the following templates with `{{ 'KEY' | translate }}`:
    - `src/app/Core/header-component/header-component.html`
    - `src/app/Core/login-component/login-component.html`
    - `src/app/Core/register-component/register-component.html`
    - `src/app/Core/forgot-password-component/forgot-password-component.html`
    - `src/app/Core/reset-password-component/reset-password-component.html`
    - `src/app/Components/home-component/home-component.html`
    - `src/app/Components/about-us-component/about-us-component.html`
    - `src/app/Components/blog-component/blog-component.html`
    - `src/app/Components/upload-image-component/upload-image-component.html`
- [ ] Replace hardcoded strings in TypeScript files (e.g., alerts, simulated API responses) using `TranslateService.instant()`.

### 3. Language Switching Logic
- [ ] Update `HeaderComponent` to inject `TranslateService`.
- [ ] Implement a `changeLanguage(lang: string)` method that:
    - Calls `this.translate.use(lang)`.
    - Sets `document.documentElement.dir` to `'rtl'` for Arabic and `'ltr'` for English.
    - Sets `document.documentElement.lang` to the selected language code.
- [ ] Bind the language selection dropdown in `header-component.html` to the `changeLanguage` method.

### 4. RTL Layout Adjustments
- [ ] Audit all templates for positional classes (`left`, `right`, `ml-`, `mr-`, `pl-`, `pr-`).
- [ ] Replace them with logical equivalents:
    - `left` $\rightarrow$ `start`
    - `right` $\rightarrow$ `end`
    - `ml-` (margin-left) $\rightarrow$ `ms-` (margin-start)
    - `mr-` (margin-right) $\rightarrow$ `me-` (margin-end)
    - `pl-` (padding-left) $\rightarrow$ `ps-` (padding-start)
    - `pr-` (padding-right) $\rightarrow$ `pe-` (padding-end)
- [ ] Adjust animations (e.g., `animate-fade-in-left` $\rightarrow$ `animate-fade-in-start`) or ensure they behave correctly in RTL mode.

## Verification Plan
- [ ] Verify that the language selector updates the text across all pages.
- [ ] Verify that selecting Arabic changes the page direction to RTL.
- [ ] Check for layout regressions in Arabic mode (e.g., overlapping elements, incorrect alignment).
- [ ] Verify that the application still functions correctly (login, registration, upload).
