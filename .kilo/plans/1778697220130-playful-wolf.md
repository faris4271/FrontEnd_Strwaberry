# Password Validation Implementation Plan

Update password validation requirements to ensure stronger passwords across the application.

## Requirements
- Minimum length: 8 characters
- At least one uppercase letter
- At least one number
- At least one special character (e.g., @, #, $, etc.)

## Implementation Steps

### 1. Define Password Validation Pattern
Create a regular expression to enforce the complexity requirements:
- Pattern: `^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@#$!%*?&])[A-Za-z\d@#$!%*?&]{8,}$`

### 2. Update Registration Component
- Modify `src/app/Core/register-component/register-component.ts`
- Update `password` field validators in `registerForm`:
    - Change `Validators.minLength(6)` to `Validators.minLength(8)`
    - Add `Validators.pattern` with the complexity regex.

### 3. Update Reset Password Component
- Modify `src/app/Core/reset-password-component/reset-password-component.ts`
- Update `newPassword` field validators in `resetPasswordForm`:
    - Change `Validators.minLength(6)` to `Validators.minLength(8)`
    - Add `Validators.pattern` with the complexity regex.

### 4. Update Login Component
- Modify `src/app/Core/login-component/login-component.ts`
- Update `password` field validators in `loginForm`:
    - Change `Validators.minLength(6)` to `Validators.minLength(8)` to maintain consistency.

### 5. Update Translation Files
Update error messages to reflect the new 8-character and complexity requirements.

- **`src/assets/i18n/en.json`**:
    - Update `register.error.password`: "Password must be at least 8 characters and include an uppercase letter, a number, and a special character (@, #, $, etc.)"

- **`src/assets/i18n/ar.json`**:
    - Update `register.error.password`: "يجب أن تكون كلمة المرور 8 أحرف على الأقل وأن تحتوي على حرف كبير، ورقم، ورمز خاص (@، #، $، إلخ)"

## Verification
- Try registering with a password that is too short (< 8).
- Try registering with a password missing a capital letter.
- Try registering with a password missing a number.
- Try registering with a password missing a special character.
- Verify that all the above trigger the validation error.
- Verify that a valid password (e.g., `Password123!`) allows form submission.
- Repeat tests for the Reset Password form.
