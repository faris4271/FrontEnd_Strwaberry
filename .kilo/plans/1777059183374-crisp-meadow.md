# Plan: Validate-Token on Page Refresh

## Current Behavior
On page refresh, `AuthInitializerService` always calls `refreshToken()` directly, even if the access token is still valid.

## Desired Behavior
On page refresh:
1. Call `/Api/V1/Authentication/Validate-Token` with the access token
2. If "NotExpired" → token is valid, no refresh needed
3. If 401 (unauthorized) → token expired, call refresh token endpoint with both access & refresh tokens

---

## Changes

### 1. `src/app/Services/auth.service.ts` — Add `validateToken()` method
- Add `validateToken()` that sends a **POST** to `/Api/V1/Authentication/Validate-Token`
- The `auth.interceptor.ts` will automatically attach the Bearer token
- Request body: `{ accessToken: string }`
- Response is `ApiResponse<string>` where `data` is `"NotExpired"` if valid
- Return `Observable<boolean>` — true if valid, false if 401

### 2. `src/app/Services/auth-initializer.service.ts` — Update initialization flow
- First call `validateToken()`:
  - If valid (NotExpired) → resolve true immediately, no refresh needed
  - If invalid (401) and refresh token exists → call `refreshToken()` 
  - If invalid and no refresh token → clear tokens, resolve false
- If no access token exists at all → resolve false

### 3. `src/app/Interceptors/refresh-token.interceptor.ts` — Exclude validate-token from auto-refresh
- Add `/Validate-Token` to the URL exclusion check so the interceptor doesn't auto-handle 401s from the validate endpoint
- Fix existing bug: `/refresh-token` exclusion is case-sensitive but actual URL is `/Refresh-Token` — update to use case-insensitive check or match the correct casing

### 4. No model changes needed
- The endpoint returns `ApiResponse<string>` — existing `ApiResponse` model is sufficient

---

## Flow After Changes

```
Page Refresh
  ↓
APP_INITIALIZER → AuthInitializerService.initialize()
  ↓
Has access token? ──No──→ resolve(false)
  ↓ Yes
validateToken() → GET /Api/V1/Authentication/Validate-Token (Bearer token attached by interceptor)
  ↓
Response data === "NotExpired"? ──Yes──→ resolve(true) ✓
  ↓ No (401)
Has refresh token? ──No──→ clearTokens(), resolve(false)
  ↓ Yes
refreshToken() → POST /Api/V1/Authentication/Refresh-Token (with accessToken + refreshToken)
  ↓
Success? ──Yes──→ resolve(true) ✓
  ↓ No
clearTokens(), resolve(false)
```
