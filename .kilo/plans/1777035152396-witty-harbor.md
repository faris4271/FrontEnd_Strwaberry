# Plan: Update Register Request and HTML Form

## Current Issues Identified

### 1. AuthService API URL Configuration (Critical)
**File:** `src/app/Services/auth.service.ts`

**Problem:** Line 17 sets `apiUrl` to a specific endpoint path:
```typescript
private apiUrl = `${environment.apiUrl}/api/ApplicationUser/Api/V1/User/Create`;
```

This is incorrect because:
- The `login()` method tries to use `${this.apiUrl}/login` which would resolve to `/api/ApplicationUser/Api/V1/User/Create/login` (wrong)
- The `refreshToken()` method tries to use `${this.apiUrl}/refresh-token` (wrong)
- The `register()` method tries to use `${this.apiUrl}/register` (wrong)

**Solution:** Change `apiUrl` to the authentication controller base path:
```typescript
private apiUrl = `${environment.apiUrl}/api/Authentication`;
```

### 2. RegisterRequest Interface Casing (Critical)
**File:** `src/app/Models/auth.models.ts`

**Problem:** Line 7 has `FullName` (capital F) which doesn't match TypeScript conventions and may not match the C# model serialization:

```typescript
export interface RegisterRequest {
  FullName: string;  // Should be fullName
  userName: string;
  email: string;
  password: string;
  confirmPassword: string;
}
```

**Solution:** Update to match C# model property names as serialized to JSON (assuming camelCase):
```typescript
export interface RegisterRequest {
  fullName: string;
  userName: string;
  email: string;
  address?: string;
  country?: string;
  phoneNumber?: string;
  password: string;
  confirmPassword: string;
}
```

**Note:** This should also match `CreateUserRequest` in `user.models.ts` to avoid duplication.

### 6. Model Duplication (Low)
**Files:** `auth.models.ts` and `user.models.ts`

**Problem:** `RegisterRequest` in `auth.models.ts` and `CreateUserRequest` in `user.models.ts` define similar structures.

**Solution:** Remove `RegisterRequest` from `auth.models.ts` and update `auth.service.ts` to import `CreateUserRequest` from `user.models.ts` instead.
