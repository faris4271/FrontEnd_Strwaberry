# Implement Photo Upload and Treatment Service

The goal is to replace the simulated photo analysis in `UploadImageComponent` with a real service that interacts with the backend API to detect diseases and retrieve treatments.

## Tasks

### 1. Define Photo Models
- Create `src/app/Models/photo.models.ts` to define the following interfaces:
    - `AnalysisResult`: containing status (normal/disease), confidence, and potentially a disease identifier.
    - `TreatmentResult`: containing the treatment plan or recommendations.

### 2. Create Photo Service
- Create `src/app/Services/photo.service.ts`.
- Inherit from `ApiService`.
- Implement `uploadPhoto(file: File)` method:
    - Use `FormData` to send the file to the backend.
    - Return an observable of `ApiResponse<AnalysisResult>`.
- Implement `getTreatment(diseaseId: string)` method:
    - Call the backend API to retrieve treatment information for the given disease.
    - Return an observable of `ApiResponse<TreatmentResult>`.

### 3. Integrate Service into UploadImageComponent
- Inject `PhotoService` into `UploadImageComponent`.
- Update `processFile()` to store the `File` object for later upload.
- Update `analyzeImage()` to:
    - Call `photoService.uploadPhoto(file)`.
    - If the result indicates a disease, call `photoService.getTreatment(diseaseId)`.
    - Update `analysisResult` and display the final outcome (Normal or Disease + Treatment).
    - Handle errors and update `errorMessage`.

### 4. Validation
- Ensure that the `UploadImageComponent` correctly transitions between loading, analysis, treatment retrieval, and result/error states.
- Verify that `FormData` is correctly constructed and sent.
- Verify that the treatment API is called only when a disease is detected.
