# Fix Translation Issue in Home Component

The Home component is currently displaying translation keys (e.g., `home.hero.title`) instead of the actual translated text. This is likely due to the translation service not having a default language set on application startup and potentially a misconfiguration in how the translation loader is provided.

## Analysis
- `HomeComponent` correctly uses `TranslateModule` and the `translate` pipe.
- Translation files `en.json` and `ar.json` contain the required keys.
- `app.config.ts` uses `TranslateModule.forRoot()` without configuration and calls `provideTranslateHttpLoader`.
- No default language is initialized when the application starts. The `HeaderComponent` allows changing the language, but nothing sets the initial language.

## Proposed Changes

### 1. Update Translation Configuration
- Review and potentially modify `src/app/app.config.ts` to ensure `TranslateModule` is correctly configured with the loader.
- If `provideTranslateHttpLoader` is insufficient or misplaced, revert to the standard `TranslateModule.forRoot({ loader: ... })` pattern using a factory.

### 2. Initialize Default Language
- Implement a way to set the default language on app startup.
- **Option A:** Update `src/app/app.ts` (the root component) to inject `TranslateService` and call `setDefaultLang('en')` and `use('en')` in the constructor.
- **Option B:** Create a new `TranslationInitializerService` and add it to the `APP_INITIALIZER` providers in `app.config.ts`.
- **Option C:** Integrate language initialization into the existing `AuthInitializerService`.

I recommend **Option A** for simplicity or **Option B** for a cleaner separation of concerns. I will evaluate which is most appropriate for the codebase.
