# Password Reset Functionality Implementation Plan

## Overview
Implement the password reset flow in the application, connecting the frontend to the `SendResetPasswordCode` and `ResetPassword` API endpoints. The reset flow is simplified to a 2-step process.

## Steps

### 1. Model Definitions
- Define interfaces for the requests in `src/app/Models/auth.models.ts`:
  - `SendResetPasswordRequest` (email)
  - `ResetPasswordRequest` (email, code, newPassword)

### 2. Service Implementation (`src/app/Services/auth.service.ts`)
- Add methods:
  - `sendResetPasswordCode(email: string): Observable<ApiResponse<void>>`
  - `resetPassword(request: ResetPasswordRequest): Observable<ApiResponse<void>>`

### 3. UI/Component Implementation
- Update `ForgotPasswordComponent` to handle a 2-step flow:
  1. Input Email -> Send Code
  2. Input Code + New Password -> Reset Password (validates and resets in one request)

### 4. Routing
- Routes for the password reset flow (e.g., `/forgot-password`).

### 5. Verification
- Run build and linting.
- Manual test the flow.
