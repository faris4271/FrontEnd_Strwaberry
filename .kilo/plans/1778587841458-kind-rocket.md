# Plan: Optimize Dockerfile with Alpine and Nginx Configuration

The goal is to provide a robust Docker setup using Alpine Linux images for the Angular application, ensuring that client-side routing works correctly and the build paths are accurate.

## 1. Confirm Alpine Usage
The current `Dockerfile` already uses Alpine-based images (`node:20-alpine` and `nginx:alpine`). This is ideal for minimizing image size.

## 2. Fix Build Output Path
Based on the project structure, the Angular build output is located at `dist/FrontEnd_Strwaberry/browser`. The `Dockerfile` will be updated to use this correct path.

## 3. Add Nginx Configuration for SPA
Angular is a Single Page Application (SPA). If a user refreshes the page on a sub-route (e.g., `/login`), Nginx will return a 404 unless configured to redirect all requests to `index.html`.
- Create a `nginx.conf` file.
- Update `Dockerfile` to copy this configuration.

## 4. Implementation Steps
1. Create `nginx.conf` in the root directory.
2. Update `Dockerfile` with the correct `COPY` path and Nginx configuration.
