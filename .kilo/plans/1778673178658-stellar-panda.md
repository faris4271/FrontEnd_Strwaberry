# Update Photo Upload and Treatment Service (Based on Actual API Response)

The initial implementation was based on an assumed API structure. The actual prediction API returns a direct response without an `ApiResponse` wrapper and uses different fields. This plan corrects the models, service, and component to align with the actual API behavior.

## API Specifications
- **Prediction Endpoint**: Returns `{ "filename": string, "prediction": string }`.
- **Healthy State**: The `prediction` field equals `"Normal"`.
- **Disease State**: The `prediction` field equals the disease name (e.g., `"Anthracnose Fruit Rot"`).
- **Treatment Endpoint**: A separate API endpoint exists to fetch treatment info based on the disease name.

## Tasks

### 1. Update Photo Models
- Modify `src/app/Models/photo.models.ts`:
    - Update `AnalysisResult` interface:
        - Remove `status`, `confidence`, `diseaseId`, and `diseaseName`.
        - Add `filename: string`.
        - Add `prediction: string`.

### 2. Update Photo Service
- Modify `src/app/Services/photo.service.ts`:
    - Update `uploadPhoto(file: File)` method:
        - Change return type to `Observable<AnalysisResult>` (removing the `ApiResponse` wrapper).
        - Ensure it calls the correct prediction endpoint.
    - Verify `getTreatment(diseaseName: string)` method:
        - Ensure it uses the prediction string as the identifier to fetch treatment data.

### 3. Integrate Updated Service into UploadImageComponent
- Modify `src/app/Components/upload-image-component/upload-image-component.ts`:
    - Update `DisplayAnalysisResult` interface to remove `confidence`.
    - Update `analyzeImage()` method:
        - Handle the direct `AnalysisResult` response from `photoService.uploadPhoto`.
        - If `analysis.prediction === 'Normal'`, set the display result to "Healthy" with a standard recommendation.
        - If `analysis.prediction !== 'Normal'`, treat the `prediction` value as the disease name and call `photoService.getTreatment(analysis.prediction)`.
        - Map the `TreatmentResult` to the `DisplayAnalysisResult` format.
        - Update error handling for both the prediction and treatment API calls.

### 4. Validation
- Verify that image uploads correctly identify "Normal" as healthy.
- Verify that disease predictions trigger the treatment API call.
- Ensure that the UI correctly displays the disease name and the retrieved treatment plan.
