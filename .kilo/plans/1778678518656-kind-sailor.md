# Add Upload Progress Bar to Image Upload

## Problem
The user wants a progress bar to be displayed when uploading a photo to the analysis server, providing visual feedback on the upload progress.

## Solution
Modify the `PhotoService` to report upload progress and update `UploadImageComponent` to track and display this progress in the UI.

## Steps

### 1. Update `PhotoService` to report progress
- Modify `uploadPhoto` method in `src/app/Services/photo.service.ts`:
    - Add `reportProgress: true` and `observe: 'events'` options to the `http.post` call.
    - Update the return type to `Observable<HttpEvent<AnalysisResult>>`.
    - Import `HttpEvent` from `@angular/common/http`.

### 2. Update `UploadImageComponent` to track progress
- Modify `src/app/Components/upload-image-component/upload-image-component.ts`:
    - Add a property `uploadProgress: number = 0`.
    - Update `analyzeImage` method:
        - Handle the stream of `HttpEvent` objects.
        - When `event.type === HttpEventType.UploadProgress`, calculate `uploadProgress = Math.round((100 * event.loaded) / (event.total || 1))`.
        - When `event.type === HttpEventType.Response`, extract the `AnalysisResult` from `event.body` and handle it as before.
        - Call `this.cdr.detectChanges()` upon progress updates.
    - Reset `uploadProgress` to `0` in `finalize` and `resetUpload`.

### 3. Update `UploadImageComponent` UI to show progress bar
- Modify `src/app/Components/upload-image-component/upload-image-component.html`:
    - In the `*ngIf="isAnalyzing"` section, add a progress bar component.
    - Use Tailwind CSS to style the progress bar (e.g., a background track and a filling bar based on `uploadProgress`).
    - Display the percentage text (e.g., "Uploading... 45%").
    - Keep or integrate with the existing spinner.

### 4. Verification
- Run `npm run build` to ensure no type errors.
- (Manual test) Upload a large image to verify the progress bar moves.
