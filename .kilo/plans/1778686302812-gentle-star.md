# Fix Navbar Mobile View Content Visibility

The navbar in mobile view does not show content when the menu is toggled. This is caused by the use of a missing animation trigger `[@slideInOut]` in `HeaderComponent` and the absence of animation providers in the application configuration.

## Changes

### 1. Update Application Configuration
- Modify `src/app/app.config.ts` to include `provideAnimations()` from `@angular/platform-browser/animations`. This enables the Angular animation system.

### 2. Implement Navbar Animations
- Modify `src/app/Core/header-component/header-component.ts`:
  - Import `trigger`, `state`, `style`, `transition`, and `animate` from `@angular/animations`.
  - Add the `animations` array to the `@Component` decorator.
  - Define the `slideInOut` trigger to handle the entry and exit of the mobile menu (e.g., sliding down from top).

## Verification
- Run the application.
- Switch to mobile view (resize browser or use dev tools).
- Click the menu toggle button.
- Verify that the mobile menu content appears and disappears with the intended animation.
