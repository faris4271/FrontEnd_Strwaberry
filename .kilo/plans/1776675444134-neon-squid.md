# Authentication Service Implementation Plan

## Overview
Implement a complete authentication service for Angular 21 with:
- Login with email/password
- Access token + Refresh token mechanism
- Memory-based token storage (secure)
- HTTP interceptors for token injection
- Route guards for protected routes

## Backend Integration
- **Base URL**: Environment-based configuration
- **Login**: `POST /api/auth/login`
- **Refresh**: `POST /api/auth/refresh-token`
- **Login Request Body**: `{ email: string, password: string }`
- **Response Format**:
  ```json
  {
    "accessToken": "jwt_token",
    "refreshToken": {
      "userName": "user@example.com",
      "tokenString": "refresh_token",
      "expireAt": "2026-04-20T10:00:00"
    }
  }
  ```

## Files to Create

### 1. Models (`src/app/Models/`)
- `auth.models.ts` - Interfaces for login request/response, refresh token

### 2. Services (`src/app/Services/`)
- `auth.service.ts` - Main authentication service
- `token.service.ts` - Token storage and management (memory-based)
- `api.service.ts` - Base API service with HTTP client

### 3. Interceptors (`src/app/Interceptors/`)
- `auth.interceptor.ts` - Attach access token to requests
- `refresh-token.interceptor.ts` - Handle 401 errors and token refresh

### 4. Guards (`src/app/Guards/`)
- `auth.guard.ts` - Protect routes requiring authentication

### 5. Environment Configuration
- `src/environments/environment.ts` - Development environment
- `src/environments/environment.prod.ts` - Production environment

## Implementation Steps

### Step 1: Environment Configuration
Create environment files with API base URL configuration.

### Step 2: Auth Models
Define TypeScript interfaces matching backend response structure.

### Step 3: Token Service
- Store access token in memory (not localStorage)
- Store refresh token string only
- Provide methods: `setTokens()`, `getAccessToken()`, `getRefreshToken()`, `clearTokens()`, `hasValidToken()`

### Step 4: Auth Service
- `login(email, password)` - Call login API, store tokens
- `refreshToken()` - Call refresh API, update tokens
- `logout()` - Clear tokens, redirect to login
- `isAuthenticated()` - Check if user is logged in
- `getCurrentUser()` - Get user info from token

### Step 5: HTTP Interceptors
- **AuthInterceptor**: Add `Authorization: Bearer <token>` header to outgoing requests
- **RefreshTokenInterceptor**: Catch 401 errors, attempt refresh, retry failed request

### Step 6: Auth Guard
Implement `CanActivateFn` guard to check authentication state before allowing route access.

### Step 7: Update App Config
Register HTTP client, interceptors, and provide environment configuration.

### Step 8: Update Login Component
Connect LoginComponent to AuthService with form handling.

## Token Refresh Flow
1. Request fails with 401
2. Interceptor catches error
3. If refresh token exists, call refresh endpoint
4. On success: store new tokens, retry original request
5. On failure: logout user, redirect to login
