# Update getTreatment to Send Current Language

The `getTreatment` method in `PhotoService` currently only sends the disease name. To support localized treatment information, it needs to send the current application language as a query parameter.

## Changes

### 1. Update `PhotoService`
- Modify `src/app/Services/photo.service.ts`:
  - Import `TranslateService` from `@ngx-translate/core`.
  - Inject `TranslateService` into the constructor.
  - Update `getTreatment(diseaseName: string)` to append the current language to the request URL using `this.translate.currentLang`.
  - The updated URL should look like: `.../get-treatment?disease=${diseaseName}&lang=${this.translate.currentLang}`.

## Verification
- Run the application.
- Change the language to Arabic.
- Upload an image that triggers a disease detection.
- Verify (via Network tab in DevTools) that the request to `get-treatment` includes `&lang=ar`.
- Repeat for English and verify `&lang=en`.
