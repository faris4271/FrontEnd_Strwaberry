# Fix Error Message Display in RegisterComponent

The user reported that the error message from the server is not being displayed when registration fails (e.g., "Email is Already Exist"). Instead, the generic Angular `HttpErrorResponse` message is shown.

## Analysis
In `RegisterComponent.onSubmit()`, the error handler uses `error.message`, which for an `HttpErrorResponse` contains the generic "Http failure response for..." message. The actual server response is contained within the `error` property of the `HttpErrorResponse` object.

## Changes
- Inject `ToastrService` into `RegisterComponent` to replace `alert()` calls for better UX and consistency with other components.
- Update the error handling logic in `onSubmit()` to prioritize `error.error?.message` over `error.message`.
- Replace all `alert()` calls in `RegisterComponent` with appropriate `this.toastr` methods.

## Implementation Details
- In `RegisterComponent`:
    - Add `private toastr = inject(ToastrService);`.
    - Replace `alert(this.translate.instant('register.error.passwordMismatch'))` with `this.toastr.error(...)`.
    - Replace `alert(this.translate.instant('register.success'))` with `this.toastr.success(...)`.
    - Update `next` callback: use `this.toastr.error` and handle `response.message`.
    - Update `error` callback: use `this.toastr.error` and extract message from `error.error?.message || error.message`.
